Adding ONTs to the GPON System on the OLT based on the profile

Created: 2022-12-01 07:17:52Latest reply: 2023-02-02 11:21:26
67441 33 0 0
View the author0#

Hi everyone,

 

Last time, I wrote a post about configuring OLT profiles. This time, I will share how to add an ONT to the GPON system.

 

You can register an ONT with a GPON port in either of the following ways:

 

Add an ONT through the ONT auto-find

 

If the ONT is automatically discovered by the registered port, you only need to confirm the information and add the ONT to the profile.

 

Access the port and enable the ONT auto-find function.

 

 

Querying the ONTs that are automatically discovered by the port. Confirm the SN of the ONT and use the SN to add the ONT to go online.

 

 

Run the ont confirm command to add the ONT to the profile by using the ONT SN, and set the management mode to OMCI. 

 

The command format is as follows:

 

ont confirm portid [ ontid ontid ] sn-or-password-auth sn-value password { password-value | hex password-value } omci [ [ ont-lineprofile-id profile-id | ont-lineprofile-name profile-name ] | [ ont-srvprofile-id profile-id | ont-srvprofile-name profile-name ]

 

 

Manually add an ONT to the GPON system

 

For an ONT that cannot be automatically discovered, you need to confirm the ONT information, and then manually add the ONT to the profile.

 

The command: ont add

 

In this post, the input parameters for adding an ONT are as follows: portid, ontid, ONT SN, ONT management mode, and the profiles to which the ONT is added.(The ontid must be unique. The ID added here is 40. The method of adding data is not only through SN, OMCI, and profiles. This section is only for demonstration.)

 

So need to confirm the ONT SN. 

 

 

And then run the following command to add the ONT SN:

 

The command format is as follows:

 

ont add portid [ ontid ontid ] sn-or-password-auth sn-value password { password-value | hex password-value } omci [ [ ont-lineprofile-id profile-id | ont-lineprofile-name profile-name ] | [ ont-srvprofile-id profile-id | ont-srvprofile-name profile-name ]

 

 

In addition to SN, OMCI, and profiles you can also use other parameters to add:

 

password/password-auth/SNMP/ONT type(Of course, you need to know the parameter information or have the corresponding configuration. The following parameters can also be used in the ont confirm command.)

 

password: Indicates the password(key) of the ONT, consisting of character strings.

 

password-auth: Two authentication modes are supported: password (key) authentication and SN+password authentication.

 

SNMP: If the ONU is managed as an independent NE and all configurations of the ONU are managed by the NMS through the SNMP protocol, select the SNMP management mode.

 

ONT type: Indicates the ONT NNI type, which means the downstream/upstream rates. (Values can be entered: 10G/10G, 10G/2.5G, 2.5G/1.25G, and auto. Default value: auto.)

 

If you only need to add data to the protection side, the command format is as follows:

 

ont add portid [ ontid ] protect-side

 

Take the configuration in this post as an example: ont add 0 40 protect-side

 

Ps: This is for demonstration purposes only.

 

Verifying the Addition

 

Run the display ont info command to query the information about the added ONT.

 

 

 

I hope this post is helpful to you. Welcome to leave a message for discussion~

Thanks for reading.
